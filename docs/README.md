# Docs Index

This directory follows a lowercase kebab-case naming convention for active docs and dated notes.

## Folders

- `plans/` - source-of-truth planning docs
- `notes/` - dated handoff notes, follow-ups, and implementation guidance
- `reports/` - running stage reports and audits
- `reference/` - stable technical reference material
- `superpowers/specs/` - design specs and deeper implementation notes

## Key Docs

- `plans/project-master-plan.md` - full project scaffold, ownership, and delivery plan
- `plans/feature-2-plan.md` - current Team 2 status and remaining handoff work
- `plans/feature-tab-mapping.md` - Q360 tab ownership and route mapping
- `reference/api-structure.md` - Q360 API and platform reference
- `reports/part-3-stage-report.md` - running Team 3 stage report
- `notes/part-3-implementation-notes.md` - Team 3 implementation guidance

## Naming Convention

- Use lowercase kebab-case for active docs.
- Keep dated notes in `YYYY-MM-DD-description.md` form.
- Prefer descriptive filenames over generic names like `report.md` or `part3.md`.
